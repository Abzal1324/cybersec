# cybersec
# 🛡️ Мой путь в Кибербезопасность: База и Практика

Привет! В этом репозитории я документирую свой процесс изучения информационной безопасности с самого нуля. Моя цель — заложить железобетонный фундамент (сети, Linux, основы уязвимостей) и подкреплять каждый шаг детальными отчетами (write-ups) и шпаргалками.

---

## 🗺️ Дорожная карта и Трекер Прогресса

### 🛑 Этап 1: Фундамент (Linux & Сети)
*Цель: уверенно чувствовать себя в консоли и понимать, как ходят пакеты по сети.*

- [ ] **Linux CLI (Командная строка)**
  - [ ] Освоить навигацию и управление файлами (`cd`, `ls`, `mkdir`, `rm`, `cp`, `mv`)
  - [ ] Разобраться с правами доступа (`chmod`, `chown`) и пользователями
  - [ ] Научиться искать информацию (`grep`, `find`, `awk`)
  - [ ] **Практика:** Пройти [OverTheWire: Bandit](https://overthewire.org/wargames/bandit/) до 15 уровня
  - [ ] *Документация:* Выложить шпаргалку по CLI и отчеты по Bandit в папку `/linux`

- [ ] **Компьютерные сети & Анализ трафика**
  - [ ] Разобраться со стеком TCP/IP и моделью OSI
  - [ ] Понять работу базовых протоколов (СLIENT-SERVER, DNS, HTTP/HTTPS, DHCP, ICMP)
  - [ ] Изучить назначение популярных портов (22, 53, 80, 443, 8080)
  - [ ] **Практика:** Установить Wireshark, перехватить свой трафик и разобрать структуру TCP-пакета
  - [ ] *Документация:* Создать конспект по сетевым протоколам в папке `/networks`

---

### 👾 Этап 2: Погружение в CTF & Уязвимости
*Цель: понять логику хакинга и защиты, познакомиться с основными категориями заданий.*

- [ ] **Знакомство с категориями CTF (Jeopardy)**
  - [ ] Разобрать теорию: что такое Web, Reverse, Pwn, Crypto, Forensics, OSINT
  - [ ] Изучить базу OWASP Top 10 (самые частые уязвимости веб-приложений)
- [ ] **Инструментарий новичка**
  - [ ] Развернуть Kali Linux (или Parrot OS) на виртуальной машине
  - [ ] Базово освоить сканер портов Nmap (флаги, логика работы)
- [ ] **Первая серьезная практика**
  - [ ] Зарегистрироваться на [picoCTF](https://picoctf.org/) и решить первые 10-15 заданий из категории *General Skills* и *Web*
  - [ ] Начать ветку Starting Point на [Hack The Box](https://www.hackthebox.com/) или легкие комнаты на [TryHackMe](https://tryhackme.com/)
  - [ ] *Документация:* Написать первые подробные райтапы (разборы) решений в папку `/ctf-writeups`

---

### 🚀 Этап 3: Закрепление и Автоматизация
*Цель: начать писать свои инструменты и анализировать чужие подходы.*

- [ ] Изучить основы Bash-скриптинга (автоматизация рутинных команд в Linux)
- [ ] Написать свой первый простейший сканер портов на Python
- [ ] Проанализировать и задокументировать одну реальную уязвимость (разбор CVE)

---

## 📂 Структура репозитория

* `/linux` — отчеты по варгеймам (Bandit) и полезные скрипты/шпаргалки по Linux.
* `/networks` — лабораторные работы с Wireshark, схемы и конспекты по сетям.
* `/ctf-writeups` — пошаговые разборы тасков с picoCTF, Hack The Box и других платформ.
